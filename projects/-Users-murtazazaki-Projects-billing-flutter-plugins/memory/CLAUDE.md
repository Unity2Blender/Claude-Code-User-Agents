# Memory Module

`MEMORY.md` is the active decision kernel. Treat `feedback_*.md` and `project_*.md` files as evidence archives unless `MEMORY.md` links them as active doctrine.

## Rules

- Current user instruction wins over all stored memories.
- Active kernel rules win over archived project or feedback files.
- Do not add runtime flags, kill switches, remote-config branches, or client toggles for UI/UX changes unless the user explicitly asks.
- Delete zero-consumer source orphans immediately; do not create sunset clocks for dead code.
- Use archive files for subsystem debugging and incident context, not as default planning doctrine.

## Editing

- Keep `MEMORY.md` compact and P0/P1 focused.
- Move long incident narratives into specific archive files.
- Mark superseded or contextual files near the top so search results do not revive stale doctrine.
