P0 HARD — Do not migrate `microservices/tutorials/` to GraphQL on Cloudflare Workers (whether Supabase `pg_graphql` proxied or Workers-hosted GraphQL gateway). Production tutorials APIs stay on REST/RPC + KV. Any future GraphQL surface must be admin-only, read-only, persisted-query-only, bearer-gated, introspection-disabled, parity-proven, and sunset-bounded.

**Why:** Audited 2026-05-03 across 6 lenses (Worker survey, Supabase schema, ADR doctrine, devops, architecture, and web research). Conclusion converged: SQL owns the correctness contracts (graph traversal, banner placement filtering, search, ACLs); KV cache keys are route-owned and route-specific (`banner:v3:*`, `search:v1:*`, etc.); the Worker is the service-role containment boundary that keeps Supabase MAU = 0. GraphQL adds 150–280 KB Worker bundle, 3–8 ms parse/validate CPU per request (pushes hot routes near the 10 ms free-tier ceiling), explodes cache cardinality 5–10× (operation-hash keys break per-resource invalidation pattern BANNER-007), reflects DB shape (contract drift on column rename), and ships none of the depth/complexity/rate guardrails GraphQL needs in prod (#1 OWASP DOS vector). The "we use ltree for reels" framing is false: ltree is HSN-only at HEAD; `tutorial_reel_edges` is a bigint DAG with cycle-prevention triggers, and `get_reel_recommendation_window_v1` is a tiered CTE union — there is no ltree-traversal benefit GraphQL would unlock. ADRs 010, 013, 015, 018 form a load-bearing core that GraphQL would conflict with or force a redesign of.

**How to apply:** When asked about GraphQL / pg_graphql / Apollo / yoga / pothos for the tutorial service, route to ADR-019 ("Tutorials APIs Stay REST/RPC; GraphQL Deferred to Gated Admin Diagnostics") and tutorials-manager `RUNBOOK-017`. Do not re-research unless one of the 6 verification checks in ADR-019 § Verification fails (then a fresh audit is required). For "too many round trips," propose a REST aggregate endpoint (canonical example: `/v1/tutorials/surface/bootstrap`), not GraphQL. For an admin diagnostic GraphQL exception to be considered, all 7 gates must be true at proposal time: (1) named cockpit/operator query gap, (2) GraphQL-vs-RPC parity proof, (3) admin-only Bearer-gated surface, (4) persisted/trusted documents only, (5) read-only, (6) service-role contained in Worker, (7) sunset clause (≤60 days, deletion not parking). Missing any gate → reject and route to "add a REST admin route" instead.
